■フォーム処理
1. 
下記フォームに入力した値をコントローラ中のメソッドで受け取る場合、そのメソッドの引数の名前として、正しいものはどれか。
なお、メソッドは入力した値を文字列型で受け取るものとする。

<input type="text" name="userName" />

1. userName
2. username
3. user
4. user_name

正答：1
解説：
『Spring Framework』第5章/パラメーター

パラメータの値（入力項目の入力値）を取得する際には、パラメータ名とメソッドの引数名を完全一致させる必要があります。大文字と小文字も区別されるため、注意してください。
したがって、選択肢1が正解です。

■フォーム処理
2.
GETリクエストとPOSTリクエストの違いとして正しいものはどれか。

1. GETリクエストでは、@RequestMappingアノテーションのmethod属性（リクエストメソッド）で、リクエスト方法を指定できる。POSTリクエストでは、method属性では指定できない。
2. GETリクエストでは、存在しないファイルにアクセスしたときに404エラーが発生する。POSTリクエストでは、404エラーが発生しない。
3. GETリクエストでは、値が日本語のリクエストパラメータを送信できない。POSTリクエストでは、日本語の値も送信できる。
4. GETリクエストでは、リクエストパラメータの内容がURL上に表示される。POSTリクエストでは、リクエストパラメータの内容は表示されない。

正答：4
解説：
『Spring Framework』第5章/GETとPOSTとは

GETリクエストとPOSTリクエストの違いは、リクエストパラメータの内容がURL上にクエリ情報として表示されるかどうかです。GETリクエストの場合は表示されますが、POSTリクエストでは表示されません。そのため、個人情報など他者に見られたくない情報を送信したい場合にはPOSTリクエストが最適です。したがって、選択肢4が正解です。
また、POSTリクエストの方が大きなサイズのデータを送信できるというメリットもあります。



■フォーム処理
3. 
Formクラスに関する説明として、誤っているものはどれか。

1. FormクラスはJavaBeansと同様にGetterメソッド、Setterメソッドを持つ。
2. パラメータの値は、一般的にパラメータ名と同名のフィールドに代入される。
3. Formクラス中には、どのコントローラで呼び出されるかを指定する処理を記述する必要がある。
4. コントローラ中のメソッドでFormクラスを受け取る際には、引数にそのための記述が必要となる。

正答：3
解説：
『Spring Framework』第5章/Formクラスの利用

Formクラス内には、基本的にフィールドとそのGetterメソッド、Setterメソッドのみを記述すればよいです。
また、Formクラスがどのコントローラで呼び出されるかは、コントローラ内のメソッドで指定します。
したがって、選択肢3が正解です。
