■ユーザと権限
1.
Oracleにおけるユーザに関する説明として正しいものはどれか。

1. 作成するユーザの名前の形式は決められており、必ず「DB_USER_番号」と付けなければならない。
2. 各ユーザは権限という情報を持っており、その権限に応じて行える操作に制限が掛かる。
3. Oracleをインストールした直後では、ユーザは1つも登録されていない。
4. 一度作成したユーザは削除することができない。

正答：2
解説：
『Oracle Database』 第3章/ユーザへの権限の付与/権限とは
Oracleでは、ユーザを自由な名前で付けることができ、作成したユーザを削除することもできます。そのため、選択肢1、4は不正解です。
また、Oracleをインストールした直後では、予め「SYS」と「SYSTEM」というデータベース管理者の権限を持ったユーザが登録されています。ユーザを新規に作成する場合はこれらのユーザでログインし、作成しましょう。
各ユーザには権限を与えなければなりません。そうしないと、作成したユーザはデータベースにログインすることすらできなくなります。一度権限を与えられたユーザは、その権限が許可する範囲のみデータベースを操作できます。したがって、選択肢2が正解です。

■ユーザと権限
2.
Oracleにおけるユーザにシステム権限を付与する構文として正しいものはどれか。

1. REVOKE 権限 FROM 対象ユーザ名;
2. GRANT システム 権限名 TO ユーザ名;
3. DROP USER ユーザ名;
4. CREATE USER ユーザ名 IDENTIFIED BY パスワード;

正答：2
解説：
『Oracle Database』 第3章/ユーザへの権限の付与/システム権限の付与
ユーザにシステム権限を与えるときにはGRANT文を使います。
したがって正解は2となります。
