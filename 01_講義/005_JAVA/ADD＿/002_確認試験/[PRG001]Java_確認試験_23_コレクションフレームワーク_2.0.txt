■コレクションフレームワーク
1.
コレクションフレームワークListと配列の違いとして正しいものはどれか。

1. Listはあらゆる型のデータを保存できるが、配列は基本型のデータのみ保存できる。
2. Listは要素数を調べる機能を持っているが、配列にはそのような機能はない。
3. Listは各要素をキーワードとなる文字列で管理するが、配列は整数値の連番で管理する。
4. Listは要素数を増減できるが、配列は要素数が固定される。

正答：4
解説：
『東京ITスクールJavaテキスト』 第26章/ArrayListクラス/配列とリストの違い
Listは、配列のように複数の値を1つの変数で管理できるインターフェイスであり、そのインターフェイスを実装したArrayListクラスが実際にその機能を実行しています。
Listは、0から始まる連番で値を管理するなど、配列との共通点が見られます。
しかし、配列の要素数が固定であるのに対して、Listは可変できるという大きな違いがあります。
開発現場ではこの柔軟性を評価され複数の値を扱う際にList、及びArrayListがよく利用されます。したがって、選択肢4が正解です。

■コレクションフレームワーク
2.下記処理の実行後、コンソールに表示される数字は次のうちどれか。

List<Integer> list = new ArrayList<>();

list.add(1);
list.add(2);
list.add(3);
list.add(4);
 
list.remove(1);

System.out.println(list.get(1));

1. 1
2. 2
3. 3
4. 4

正答：3
解説：
『東京ITスクールJavaテキスト』 第26章/ArrayListクラス/要素の削除
リストから要素を削除した場合、後続する要素は1つ手前に詰めて配置し直されます。
「3」という値はもともと添字2番の要素に代入されていました。
しかし、添字1番の要素(2)が削除されたことで、値「3」が代入された要素の添字が1番となりました。
したがって、選択肢3が正解です。
